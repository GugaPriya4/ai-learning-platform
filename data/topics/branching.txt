Branching

Definition:
A branch in Git is a lightweight movable pointer to a specific commit, allowing parallel lines of development.

Explanation:
Branching lets you diverge from the main line of development and work independently without affecting the main codebase. The default branch is called main or master. When you create a branch, Git creates a new pointer to the current commit.

Why branching matters:
- Work on new features without breaking working code
- Multiple developers work on different features simultaneously
- Experiment safely knowing you can always switch back
- Keep production code stable while developing

Real world example:
Think of branches like parallel universes. Your main branch is the stable universe where everything works. When you want to add a new feature, you create a new universe (branch) where you can experiment. If the experiment works, you merge it back. If not, you delete that universe and nothing is affected.

Common commands:
- git branch (list all branches)
- git branch feature-name (create new branch)
- git checkout feature-name (switch to branch)
- git checkout -b feature-name (create and switch in one command)
- git branch -d feature-name (delete branch)
- git branch -r (list remote branches)

Step by step example:
1. git checkout -b login-feature
2. Make changes to login files
3. git add . and git commit -m "Add login feature"
4. git checkout main
5. git merge login-feature
6. git branch -d login-feature
