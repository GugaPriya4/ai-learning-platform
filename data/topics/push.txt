Git Push

Definition:
Git push uploads commits from your local repository to a remote repository, making your changes available to others.

Explanation:
After committing changes locally, those commits only exist on your computer. Git push sends them to the remote repository on GitHub or another server. This makes your work visible to collaborators and backs it up remotely.

How push works:
Git compares your local commits with the remote repository. It identifies commits that exist locally but not remotely. It uploads only those new commits, not the entire repository. This makes push fast and efficient.

Real world example:
Think of push like submitting your assignment to Google Classroom. You have been working on your assignment on your own laptop (local commits). When you are done, you submit it on Google Classroom (git push). Now your teacher and classmates can see it. Until you submit, only you can see it.

Common commands:
- git push origin main (push main branch to remote)
- git push (push current branch if tracking is set up)
- git push -u origin main (push and set upstream tracking)
- git push origin branch-name (push a specific branch)
- git push --force (overwrite remote, use with caution)

Step by step example:
1. Make changes to files
2. Run git add .
3. Run git commit -m "Update homepage design"
4. Run git push origin main
5. Go to GitHub and refresh, your changes are there

Common errors:
- Rejected push: Remote has commits you do not have locally
  Solution: Run git pull first, then git push
- No upstream branch: First time pushing a new branch
  Solution: Run git push -u origin branch-name

Best practices:
- Always pull before you push when collaborating
- Never force push to shared branches
- Push frequently so your work is backed up
