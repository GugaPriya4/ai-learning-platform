Repository

Definition:
A Git repository is a directory that contains your project files along with the entire history of changes made to those files.

Explanation:
When you initialize a Git repository, Git creates a hidden .git folder inside your project directory. This folder contains all the metadata and object database for your project. The repository stores every version of every file you have ever committed.

Types of repositories:
- Local repository: Lives on your computer
- Remote repository: Lives on a server like GitHub, GitLab, or Bitbucket
- Bare repository: Contains only Git data, no working files (used on servers)

Key components inside .git folder:
- HEAD: Points to the current branch
- objects/: Stores all your file content as compressed objects
- refs/: Stores pointers to commits (branches and tags)
- config: Repository-specific configuration

Real world example:
Think of a repository like a filing cabinet for your project. Every document (file) goes in the cabinet. Every time you make changes and commit, Git takes a photograph of the entire cabinet and stores it. You can always go back and look at any photograph from the past.

Common commands:
- git init: Create a new repository in current directory
- git clone URL: Copy an existing remote repository to your computer
- git status: See the current state of your repository
- git log: View the history of commits in your repository

Creating a repository:
Method 1 - Start fresh:
  mkdir my-project
  cd my-project
  git init

Method 2 - From existing project:
  cd existing-project
  git init
  git add .
  git commit -m "Initial commit"

Method 3 - Clone from GitHub:
  git clone https://github.com/username/repository-name
