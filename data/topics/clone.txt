Clone

Definition:
Git clone is a command that creates a complete copy of a remote repository on your local machine including all files, branches, and commit history.

Explanation:
When you clone a repository, Git downloads everything from the remote repository to your computer. You get a fully functional local repository that is connected to the original remote repository. Any changes you make locally can be pushed back to the remote.

What cloning does:
- Downloads all files in the repository
- Downloads the complete commit history
- Creates all branches that exist on remote
- Sets up the remote connection automatically called origin
- Checks out the default branch ready to work

Real world example:
Think of cloning like photocopying an entire filing cabinet. The photocopy has every document, every folder, everything organized exactly the same way. You can work on your photocopy without affecting the original. When you are done, you can send your changes back to the original cabinet.

Common commands:
- git clone https://github.com/username/repo-name
- git clone https://github.com/username/repo-name my-folder-name
- git clone --depth 1 URL (shallow clone, only latest snapshot)

Step by step example:
1. Go to GitHub and find a repository
2. Click the green Code button and copy the URL
3. Open your terminal
4. Type: git clone https://github.com/username/repo-name
5. Git downloads everything
6. cd into the new folder and start working

Difference between clone and fork:
- Clone: Downloads repository to your local machine
- Fork: Creates your own copy on GitHub (server side)
- You usually fork first on GitHub, then clone your fork to your computer
